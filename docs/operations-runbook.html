Local Run and Production Deployment Runbook

A practical operations guide for setting up, running, validating, deploying, monitoring, and troubleshooting the Agentic Research Agent project.

Document type: Operations Runbook Generated: 2026-06-04 Project: agentic-ai-learning Runtime: Python 3.13

What This Project Runs As

This project currently ships as a Python package with a Typer command-line interface named research-agent. It can also be used as a library by importing ResearchAgent. There is no HTTP server, Dockerfile, Kubernetes manifest, or managed API deployment in the repository yet.

Supported nowLocal CLI usage, local chat, RAG ingestion, test/lint/type-check workflow, and package/library usage.
Production-ready pattern nowRun as a scheduled job, internal CLI tool, worker process, or library inside another service.
Needs extra implementationPublic API deployment, authentication, request rate limiting, container image, health endpoints, and durable conversation storage.

Local Setup From Fresh Clone

1. Enter the project directory

cd "/home/joginder-singh/Study/Agentic AI/langchain"

2. Install dependencies

uv sync

This creates or updates the local virtual environment and installs runtime plus development dependencies from pyproject.toml and uv.lock.

3. Create local environment file

cp .env.example .env

4. Configure one LLM provider

For Groq, edit .env and set:

LLM_PROVIDER=groq
LLM_MODEL=llama-3.3-70b-versatile
GROQ_API_KEY=your_real_key_here

For OpenAI:

LLM_PROVIDER=openai
LLM_MODEL=gpt-4.1-mini
OPENAI_API_KEY=your_real_key_here

For Anthropic:

LLM_PROVIDER=anthropic
LLM_MODEL=claude-sonnet-4-6
ANTHROPIC_API_KEY=your_real_key_here

For local Ollama:

ollama serve
ollama pull llama3.1
LLM_PROVIDER=ollama
LLM_MODEL=llama3.1
OLLAMA_BASE_URL=http://localhost:11434
Secret rule: Never commit .env. It is ignored by git and must stay local or come from a production secret manager.

Build the Knowledge Base

The agent can search local documents from data/knowledge_base. Supported source formats are Markdown, text, reStructuredText, and PDF.

Normal ingestion

uv run research-agent ingest

Force rebuild after changing documents

uv run research-agent ingest --force

Add new documents

cp your-document.md data/knowledge_base/
uv run research-agent ingest --force

The generated vector index is written to data/vector_store. This folder is intentionally ignored by git because it is generated data.

Run Locally

Ask one question

uv run research-agent ask "What is the ReAct pattern in agentic AI?"

Use a named conversation thread

uv run research-agent ask "Explain LangGraph memory in simple terms" --thread learning-session

Start interactive chat

uv run research-agent chat

Interactive chat with a custom thread

uv run research-agent chat --thread support-demo

Run as a Python library

uv run python - <<'PY'
from agentic_research_agent import ResearchAgent

agent = ResearchAgent()
response = agent.ask("Compute sqrt(144), then explain RAG in one sentence.")
print(response.answer)
print("run id:", response.run_id)
print("duration ms:", response.duration_ms)
print("tools:", [tool.name for tool in response.tool_calls])
PY

Production Deployment Options

The safest production path today is to deploy as a controlled CLI, batch job, or worker. An HTTP API should be added deliberately with authentication, health checks, and rate limiting.

Production Deployment: CLI or Batch Job

Use this pattern when the agent is run by trusted operators, cron, Airflow, Jenkins, GitHub Actions, or an internal automation worker.

1. Prepare host

python --version
uv --version
git --version

2. Fetch code and install

git clone <your-repository-url> agentic-research-agent
cd agentic-research-agent
uv sync --frozen

3. Provide production environment

export ENVIRONMENT=production
export LOG_LEVEL=INFO
export LLM_PROVIDER=groq
export LLM_MODEL=llama-3.3-70b-versatile
export GROQ_API_KEY="value-from-secret-manager"
export VECTOR_STORE_DIR="/var/lib/agentic-research-agent/vector_store"
export KNOWLEDGE_BASE_DIR="/opt/agentic-research-agent/knowledge_base"

4. Build or restore vector store

uv run research-agent ingest --force

For repeatable deployments, prefer building the vector store during release and restoring it as a versioned artifact or persistent volume.

5. Smoke test

uv run research-agent ask "Say hello and compute 12 * 12."

6. Schedule a job

# Example cron entry, runs daily at 08:00.
0 8 * * * cd /opt/agentic-research-agent && /usr/local/bin/uv run research-agent ask "Create today's research summary" >> /var/log/agentic-research-agent/job.log 2>&1
Production logging: Set ENVIRONMENT=production. The application emits JSON logs from the package logger, which are easier for log collectors to parse.

Production Deployment: Container Pattern

The repository does not currently include a Dockerfile. The following is a recommended pattern if you decide to containerize it.

Suggested Dockerfile

FROM python:3.13-slim

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    ENVIRONMENT=production

WORKDIR /app

RUN apt-get update \
    && apt-get install -y --no-install-recommends curl build-essential \
    && rm -rf /var/lib/apt/lists/*

RUN pip install --no-cache-dir uv

COPY pyproject.toml uv.lock README.md ./
COPY src ./src
COPY data/knowledge_base ./data/knowledge_base

RUN uv sync --frozen --no-dev

VOLUME ["/app/data/vector_store"]

ENTRYPOINT ["uv", "run", "research-agent"]

Build image

docker build -t agentic-research-agent:0.1.0 .

Run ingestion in container

docker run --rm \
  --env-file .env.production \
  -v agentic_vector_store:/app/data/vector_store \
  agentic-research-agent:0.1.0 ingest --force

Ask a question in container

docker run --rm \
  --env-file .env.production \
  -v agentic_vector_store:/app/data/vector_store \
  agentic-research-agent:0.1.0 ask "What is RAG?"
Important: Do not bake real provider keys into the image. Use runtime environment variables, Docker secrets, Kubernetes secrets, or a cloud secret manager.

Production Deployment: API Service Path

If the project must serve multiple users over HTTP, add a thin API layer around ResearchAgent. Do not expose the CLI process directly as a public service.

Minimum FastAPI shape

from fastapi import FastAPI
from pydantic import BaseModel
from agentic_research_agent import ResearchAgent

app = FastAPI()
agent = ResearchAgent()

class AskRequest(BaseModel):
    question: str
    thread_id: str = "default"

@app.get("/health")
def health():
    return {"status": "ok"}

@app.post("/ask")
def ask(request: AskRequest):
    return agent.ask(request.question, thread_id=request.thread_id).model_dump()

API production controls to add

  • Authentication and authorization for every request.
  • Rate limits per user, team, or API key.
  • Request size limits and timeout settings.
  • Durable LangGraph checkpointing instead of in-memory MemorySaver.
  • Central tracing for LLM calls, tool calls, errors, and latency.
  • Health endpoints that verify vector store readiness and provider connectivity.

Backup and Restore

What to back up

  • data/knowledge_base: source documents, the canonical RAG corpus.
  • data/vector_store: generated Chroma index, useful for faster restore.
  • Production configuration in secret/config manager, not plain text files.

Backup command example

tar -czf knowledge-base-backup-$(date +%Y%m%d).tgz data/knowledge_base

Restore command example

tar -xzf knowledge-base-backup-20260604.tgz
uv run research-agent ingest --force

If the vector store is not backed up, it can be regenerated from the knowledge base documents.

Release Checklist

  1. Confirm target provider, model, and API key are configured in the target environment.
  2. Run uv sync --frozen, uv run pytest, uv run ruff check ., and uv run mypy.
  3. Confirm knowledge base documents are current and approved for indexing.
  4. Run uv run research-agent ingest --force or restore a trusted vector store artifact.
  5. Run a smoke test question that exercises calculator and knowledge base retrieval.
  6. Verify production logs reach the log collector.
  7. Confirm rollback plan: previous code version, previous knowledge base corpus, and previous vector store artifact.
