1. Overview

The project now exposes two interchangeable agent services behind one interface (AgentService) and one selector (AGENT_MODE):

ModeWhat it isBest for
single (default)ReAct tool-using agent (web search · KB · calculator)General Q&A, tool use, calculations
multiagentSupervisor RAG pipeline: retrieval → synthesis → critic, with hybrid search + reranking + grounding checksDocument-grounded answers with citations and verification

Both share the same FastAPI surface, authentication, rate limiting, checkpointer, metrics, logging, and deployment artifacts. Switching is a config change — no code change.

2. Why Multi-Agent RAG

Separation of dutiesA supervisor routes, a retrieval agent gathers evidence, a synthesis agent writes, a critic verifies. Each is small, testable, and independently improvable.
Retrieval is the productMost RAG failures are retrieval failures. Hybrid search + reranking + multi-query expansion materially raise recall and precision over plain vector search.
Grounding & trustThe critic checks that every claim is supported by retrieved context and can trigger a bounded re-retrieval — reducing hallucination and enabling citations.
Bounded costHandoffs are explicit and revision rounds are capped (MAX_REVISIONS), so agent "chatter" can't blow up latency or tokens.

3. The Agents & Graph

Figure 1 — Implemented as a LangGraph StateGraph with conditional edges. Nodes call the LLM directly; the retriever and query expander are injected (so the graph is unit-testable with fakes).

AgentResponsibilityOutput to state
SupervisorClassify whether retrieval is needed (RETRIEVE/DIRECT); reset per-run scratch stateroute
RetrievalExpand query → hybrid search per query → fuse + cap contextqueries, documents
SynthesisDraft a grounded answer with inline [n] citations (or a direct answer)draft, answer
CriticVerify grounding; emit GROUNDED/NOT_GROUNDED + feedback; bounded loopgrounded, critique, revisions

4. Advanced Retrieval Pipeline

Figure 2 — *Reranking is optional (RERANK_ENABLED); it downloads a cross-encoder on first use. Without it, the fused top-k is used directly.

  • Multi-query expansion — the LLM rewrites the question into several phrasings; results are unioned (recall ↑). Degrades gracefully to the original query if the LLM is unavailable.
  • Hybrid search — dense (vector embeddings) + sparse (BM25) retrieve in parallel; complementary strengths (semantics vs exact terms).
  • Reciprocal Rank Fusion — combines the two rankings by rank position (robust to score scales), de-duplicating by source+content.
  • Reranking — an optional cross-encoder scores each (query, chunk) pair jointly for a precision boost before the top-k is sent to the model.
  • Critic loop — if the answer isn't grounded, the critic's feedback seeds a focused re-retrieval, bounded by MAX_REVISIONS.

│   ├── reranker.py            # Reranker protocol + CrossEncoderReranker (lazy)
│   └── query_transform.py     # QueryTransformer (multi-query, graceful fallback)
├── multiagent/                # the supervisor RAG graph
│   ├── state.py               # MultiAgentState, RetrievedChunk
│   ├── prompts.py             # supervisor / synthesis / critic prompts
│   ├── graph.py               # build_multiagent_graph (StateGraph + routing)
│   └── service.py             # MultiAgentRAG facade (ask / stream / health)
├── agents/base.py             # AgentService protocol (shared interface)
└── service_factory.py         # build_agent_service() → single | multiagent

The retriever is decoupled from any store: it takes a dense_search callable plus an in-memory corpus for BM25, so fusion is unit-testable without embeddings. build_hybrid_retriever() adapts the existing KnowledgeBase (Chroma) into those inputs.

Tuning order: (1) get chunking right, (2) enable hybrid + multi-query (on by default), (3) turn on reranking for precision, (4) raise MAX_REVISIONS only if grounding is still weak — each step trades latency/cost for quality.

7. Run & Use It

# 1. install + configure + ingest (same as single mode)
uv sync
cp .env.example .env            # set a provider key
uv run research-agent ingest

# 2. select multi-agent RAG
export AGENT_MODE=multiagent

# CLI
uv run research-agent ask "What is RAG? Cite sources."
uv run research-agent chat

# HTTP API (same endpoints, auth, metrics)
uv run research-agent-api
curl -X POST localhost:8000/v1/ask -H "X-API-Key: $API_KEYS" \
  -H "Content-Type: application/json" \
  -d '{"question":"Explain the ReAct pattern.","thread_id":"demo"}'

[retrieval] 4 chunks

<final answer with [n] citations>

8. Deployment & Scaling

Multi-agent mode runs on the same production stack as single mode — FastAPI + Uvicorn/Gunicorn, durable checkpointer, Docker/Compose/Kubernetes, Prometheus metrics, health probes. See the Production-Grade Guide for the step-by-step deploy and base scaling. RAG-specific considerations:

ConcernGuidance
Latency / cost per requestMore LLM calls per answer (supervisor + synthesis + critic + multi-query). Use a fast model for supervisor/critic; cap MULTIQUERY_COUNT and MAX_REVISIONS; cache repeated questions.
Vector storeLocal Chroma is fine for one node; for multi-replica move to a shared vector DB (pgvector/Qdrant). The build_hybrid_retriever seam isolates this swap.
BM25 indexBuilt in-process from the corpus at startup. For very large corpora, move sparse retrieval into the vector DB (e.g. Qdrant/Elasticsearch hybrid) instead of in-memory BM25.
RerankerCPU-heavy cross-encoder. Run it as a dedicated service or use a hosted reranker at scale; keep it off the hot path when latency-critical.
IngestionKeep as an out-of-band job that writes to the shared vector DB; the serving path only reads.
Scale-outThe API tier stays stateless → scale horizontally with HPA. Set maxReplicas below the LLM provider's concurrency ceiling.

9. Quality & Tests

The suite is fully offline (fake LLMs + fake retrievers — no API key, no network, no model download):

  • Retriever — RRF fusion, BM25 lexical-overlap matching, de-dup, reranker application.
  • Query transform — passthrough without an LLM, parsing of rewrites, graceful failure.
  • Multi-agent graph — retrieve path, direct path, critic-triggered revision, and revision-budget enforcement.
  • API — auth, rate limit, input cap, health, metrics (mode-agnostic via the shared protocol).
